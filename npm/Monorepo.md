# 什么是 Monorepo

## Monorepo = 单仓库多应用

在前端中，通常包括：

- Web 应用（App）
- UI 组件库（packages/ui）
- 工具库（packages/utils）

## 为什么使用？

- 代码复用：不同项目共享逻辑时，不需要额外发布 npm 包，只需要 `packages/utils` 修改一下，全局都可以使用
- 依赖管理：通过 pnpm workspace 进行依赖管理，不同项目使用相同版本
- 统一构建：可以统一构建工具，统一代码风格，统一脚本，产物统一规范
- 顺畅协作：不需要在多个项目多个包之间频繁切换仓库

## 项目发展

- 单仓库单应用：一个仓库就是一个应用，随着项目功能迭代，仓库变大难以维护
- 多仓库多应用：多个仓库分配给多个应用，无法进行项目依赖与代码的共享，结构复杂
- 单仓库多应用：一个仓库管理多个应用，业务代码分离解耦，应用间共享依赖与代码，项目扩展更方便
